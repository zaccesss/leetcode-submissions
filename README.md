## About

This repository is a long-term archive of my accepted LeetCode submissions. Every solution is pushed here automatically the moment it is accepted on the platform. I follow the NeetCode 250 roadmap in order, working through Arrays and Hashing first before moving on to Two Pointers, Sliding Window, Stack and beyond.

The goal is simple: at least one problem solved every single day.

## Structure

Each accepted submission gets its own folder named by problem number and title. LeetHub generates a README inside each folder with the full problem description, constraints and examples.

```

    solution.py
    README.md

0121-best-time-to-buy-and-sell-stock/
    solution.py
    README.md

0242-valid-anagram/
```

Problems are solved in Python by default. Java, Kotlin, C++ and other languages appear where the problem or interview context calls for it.

## Approach

Every problem follows the same method regardless of difficulty.

1. Read and understand the problem fully before writing any code.
2. Think through a brute force approach first.
3. Identify the pattern or technique (sliding window, two pointers, hash map and so on).
4. Optimise before coding where possible.
5. Write the solution, test it on edge cases then submit.

Solutions prioritise clarity and understanding over cleverness. The point is to build genuine algorithmic thinking, not to memorise answers.
